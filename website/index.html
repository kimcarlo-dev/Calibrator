Guided setup

Setup Instructions

0 of 6 complete

Follow this checklist to prepare Arduino IDE, board manager, libraries, firmware upload, and dashboard connection.

1

Install ESP32 Board Manager

In Arduino IDE, open File > Preferences and add this to Additional boards manager URLs:

https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json

  • Go to Tools > Board > Boards Manager.
  • Search for esp32 by Espressif and install version 2.0.x+.
  • Select Tools > Board > ESP32 Arduino > ESP32 Dev Module.
2

Install All Libraries

Open Sketch > Include Library > Manage Libraries. This list includes required, ESP32-bundled, and optional alternatives.

Important: optional libraries are included for completeness; the supplied firmware uses direct implementations instead. See libraries.md for usage notes and known-good versions.

3

Configure Wi-Fi on each ESP32

Open every sketch and edit the WIFI_CONFIG block at the top:

const WiFiCredentials WIFI_CONFIG = { "your-ssid", "your-password" };

  • Use a 2.4 GHz network (ESP32 does not join 5 GHz).
  • Keep DHCP enabled. The router automatically gives every ESP32 a different IP.
  • Keep the four supplied HOSTNAME values unique; the dashboard uses them even when DHCP addresses change.
  • For maximum reliability, create four DHCP reservations in the router using the MAC addresses printed in Serial Monitor.
4

Upload the Firmware

Open each .ino file from the assets/firmware/ folder:

  • esp32_1_sensors.ino
  • esp32_2_16channel_relay.ino
  • esp32_3_8channel_4channel_relay.ino
  • esp32_4_extras.ino

Select the matching serial port and click Upload. The board's IP is printed on the Serial monitor after boot.

esp32_1_sensors.ino esp32_2_16channel_relay.ino esp32_3_8channel_4channel_relay.ino esp32_4_extras.ino Download all firmware (zip)
5

Open the Dashboard

  • Open website/index.html directly, or serve it locally:
  • cd website && python -m http.server 8000
  • Navigate to http://localhost:8000.
  • The Connection tab starts with the four recommended .local hostnames. Click Connect All.
  • If your router does not support mDNS, replace each hostname with that board's unique IP from Serial Monitor.
6

Troubleshooting

  • Wrong IP? Check the Serial monitor or your router's DHCP table.
  • WebSocket fails? Serve the dashboard over HTTP, not file://.
  • Relay turns OFF? Use ON, not Pulse; Pulse is a timed test. Repeated [BOOT] messages indicate a power-related ESP32 reset.
  • Relays click but the load doesn't switch? Provide a separate 5 V supply for the relay coils.
  • Need more detail? See docs/ for the calibration guide and API reference.
